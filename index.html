Palo Alto Interview Prep

بص يا حبيبي علشان بحبك جبتلك أهم 40 سؤال (سهل، متوسط، صعب)

1) ما هو Palo Alto Firewall؟ +
هو Next Generation Firewall (NGFW) من شركة Palo Alto Networks، بيقدم تحكم مش بس على مستوى الـ IP والـ Port، لكن كمان على مستوى التطبيق (App) والمستخدم (User) والمحتوى (Content).
2) الفرق بين Traditional Firewall و NGFW؟ +
Traditional: بيشتغل Layer 3/4، بيعتمد على Port/IP، حماية محدودة ولا يوجد User Awareness. NGFW: بيشتغل Layer 7، بيعتمد على Application، يدعم User-ID، ويوفر (IPS + URL + AV + App Control).
3) مكونات Palo Alto الأساسية؟ +
● App-ID ● User-ID ● Content-ID ● Security Policies ● NAT Policies ● Security Profiles
4) ما هي الـ Security Zones؟ +
تقسيم منطقي للشبكة (Trust - Untrust - DMZ). السياسات بتطبق بين Zone و Zone مش Interface و Interface.
5) الفرق بين Security Policy و NAT Policy؟ +
● Security Policy: تسمح أو تمنع الترافيك. ● NAT Policy: تعدل الـ Source أو Destination IP.
6) الفرق بين Source NAT و Destination NAT؟ +
● SNAT: تغيير Source IP (غالباً خروج إنترنت). ● DNAT: تغيير Destination IP (مثل Port Forwarding).
7) Static NAT vs Dynamic NAT؟ +
● Static: IP مقابل IP ثابت (1:1). ● Dynamic: الـ IP بيتغير من Pool.
8) ما هو الـ App-ID؟ +
تقنية بتتعرف على التطبيق الحقيقي حتى لو غير الـ Port (مثال: Facebook على Port 443).
9) ما هو الـ User-ID؟ +
يربط الـ IP بالـ User من الـ Active Directory علشان تكتب Policy بالمستخدم مش بالـ IP.
10) ما هو الـ Content-ID؟ +
يفحص المحتوى (Virus, Spyware, URL, File Blocking).
11) Layer 2 Mode vs Layer 3 Mode؟ +
● L3: وضع الـ Routing العادي (الأكثر استخداماً). ● L2: وضع الـ Switching بدون Routing.
12) Virtual Wire (V-Wire)؟ +
بيشتغل Inline بين جهازين بدون تغيير IP — مفيد في بيئة Production قديمة.
13) Tap Mode؟ +
للمراقبة فقط (Monitoring) بدون تدخل في الترافيك.
14) Virtual Router vs Routing Table؟ +
● Virtual Router: كيان منطقي يعمل Routing. ● Routing Table: جدول المسارات داخل الـ VR.
15) ما هي الـ Security Profiles؟ +
تشمل أدوات الحماية مثل: Antivirus, Anti-Spyware, Vulnerability Protection, URL Filtering, File Blocking.
16) ما هو WildFire؟ +
خدمة Cloud من Palo Alto Networks بتحلل الملفات المشبوهة في Sandbox.
17) Application Default vs Any؟ +
● Application Default: يسمح للتطبيق على البورت الأصلي فقط. ● Any: يسمح على أي بورت (أقل أماناً).
18) ما هو الـ Service Object؟ +
تحديد Port معين (مثال: TCP 80).
19) ما هي الـ Session Table؟ +
جدول فيه كل الـ Sessions النشطة داخل الفايروول.
20) ترتيب معالجة الـ Packet داخل Palo Alto؟ +
1. Zone Check 2. NAT Policy 3. Security Policy 4. Security Profiles 5. Forwarding
21) ما هو SSL Decryption؟ +
فك تشفير HTTPS علشان الفايروول يفحص المحتوى.
22) SSL Forward Proxy vs SSL Inbound Inspection؟ +
● Forward Proxy: لفحص خروج المستخدمين للإنترنت. ● Inbound Inspection: لفحص الترافيك داخل السيرفرات.
23) DoS Protection؟ +
يحمي من الـ Flood Attacks.
24) Zone Protection؟ +
يحمي الـ Zone نفسها من: SYN Flood, Port Scan, ICMP Flood.
25) URL Filtering vs DNS Security؟ +
● URL Filtering: فلترة المواقع. ● DNS Security: يمنع DNS-based attacks و C2 domains.
26) كيف يعمل الـ App-ID بالتفصيل؟ +
● Signature Matching ● Protocol Decoding ● Heuristic Analysis ● SSL Decryption (لو مشفر) بيعمل Re-evaluation للجلسة لو التطبيق اتغير.
27) Stateful Inspection vs Application Inspection؟ +
● Stateful: يتابع حالة الاتصال (TCP Handshake). ● Application: يفهم التطبيق نفسه في Layer 7.
28) كيف يتم إنشاء الـ Session؟ +
أول Packet → Lookup NAT → Match Security Policy → Create Session → Add to Session Table.
29) كيف يعمل الـ Threat Prevention؟ +
يفحص الترافيك باستخدام: ● IPS Signatures ● Antivirus ● Anti-Spyware ● WildFire Integration
30) Security Profile vs Security Profile Group؟ +
● Profile: نوع حماية واحد. ● Profile Group: مجموعة Profiles متجمعة.
31) ما هو الـ Panorama؟ +
منصة إدارة مركزية من Palo Alto Networks لإدارة أكثر من Firewall.
32) Device Group vs Template؟ +
● Device Group: للـ Policies. ● Template: للـ Network & Interface Settings.
33) Commit vs Push؟ +
● Commit: حفظ التغييرات محلياً. ● Push: إرسالها من Panorama للأجهزة.
34) ما هو High Availability (HA)؟ +
ربط جهازين Firewall لزيادة التوافر.
35) Active/Active vs Active/Passive؟ +
● Active/Passive: واحد شغال والتاني Standby (الأكثر استخداماً). ● Active/Active: الاثنين شغالين.
36) ما هي الـ HA Interfaces؟ +
● HA1: Control Link ● HA2: Session Sync ● HA3: Packet Forwarding (A/A)
37) كيف يتم الـ Synchronization؟ +
Sync Config, Sync Session Table, Heartbeat Monitoring.
38) خطوات الـ Troubleshooting لو الترافيك مش بيمر؟ +
1. Check Interface Up/Down 2. Check Zones 3. Check NAT 4. Check Security Policy 5. Check Routing 6. Check Logs
39) أهم أوامر الـ CLI؟ +
show session all show session id <id> show routing route show interface all show log traffic test security-policy-match
40) تتبع Session معينة؟ +
1. هات Session ID من Traffic Logs. 2. CLI: show session id <session-id> 3. لو لزم الأمر Packet Capture.
