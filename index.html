Palo Alto Interview Prep

40 Questions to Master Your Interview

1) ما هو Palo Alto Firewall؟ +
هو Next Generation Firewall (NGFW) من شركة Palo Alto Networks. بيقدم تحكم مش بس على مستوى الـ IP والـ Port، لكن كمان على مستوى التطبيق (App) والمستخدم (User) والمحتوى (Content).
2) الفرق بين Traditional Firewall و NGFW؟ +
Traditional: بيشتغل Layer 3/4، بيعتمد على Port/IP، حماية محدودة.
NGFW: بيشتغل Layer 7، بيعتمد على Application، يدعم User-ID، ويوفر (IPS + URL + AV + App Control).
3) مكونات Palo Alto الأساسية؟ +
  • App-ID / User-ID / Content-ID
  • Security Policies / NAT Policies
  • Security Profiles
4) ما هي الـ Security Zones؟ +
تقسيم منطقي للشبكة مثل (Trust – Untrust – DMZ). السياسات في بالو ألتو تُطبق دائماً بين Zones وليس Interfaces.
5) الفرق بين Security Policy و NAT Policy؟ +
Security Policy: وظيفتها السماح أو المنع (Allow/Deny) للترافيك.
NAT Policy: وظيفتها تعديل الـ Source أو Destination IP فقط.
6) الفرق بين Source NAT و Destination NAT؟ +
SNAT: تغيير الـ Source IP (غالباً عند خروج المستخدمين للإنترنت).
DNAT: تغيير الـ Destination IP (Port Forwarding للسيرفرات الداخلية).
7) Static NAT vs Dynamic NAT؟ +
Static: حجز IP مقابل IP (ثابت 1:1).
Dynamic: الجهاز بيختار IP متغير من الـ Pool المتاح.
8) ما هو الـ App-ID؟ +
تقنية بتتعرف على التطبيق الحقيقي حتى لو غير الـ Port بتاعه (مثال: Facebook شغال على Port 443، الفايروول بيفهم إنه فيسبوك مش مجرد ترافيك ويب عادي).
9) ما هو الـ User-ID؟ +
بيربط الـ IP بالـ User من الـ Active Directory علشان تقدر تكتب Policy باسم المستخدم بدل ما تستخدم IP.
10) ما هو الـ Content-ID؟ +
بيفحص محتوى الـ Packet للبحث عن (Virus, Spyware, URL, File Blocking).
11) Layer 2 Mode vs Layer 3 Mode؟ +
L3: وضع الـ Routing العادي (الأكثر استخداماً).
L2: وضع الـ Switching بدون Routing.
12) Virtual Wire (V-Wire)؟ +
بيشتغل Inline بين جهازين بدون تغيير الـ IP. مفيد جداً في بيئة Production قديمة كحل شفاف (Transparent).
13) Tap Mode؟ +
للمراقبة فقط (Monitoring) بدون أي تدخل في الترافيك أو منعه.
14) Virtual Router vs Routing Table؟ +
Virtual Router: كيان منطقي بيعمل Routing داخل الفايروول.
Routing Table: هو جدول المسارات الفعلي داخل هذا الـ VR.
15) ما هي الـ Security Profiles؟ +
هي أدوات الفحص مثل: Antivirus, Anti-Spyware, Vulnerability Protection, URL Filtering, File Blocking.
16) ما هو WildFire؟ +
خدمة Cloud من Palo Alto لتحليل الملفات المشبوهة والجديدة (Zero-day) في بيئة معزولة (Sandbox).
17) Application Default vs Any؟ +
App Default: بيسمح للتطبيق فقط على البورت الأصلي بتاعه (أكثر أماناً).
Any: بيسمح للتطبيق على أي بورت (أقل أماناً).
18) Service Object؟ +
تحديد بورت معين مثل (TCP: 80).
19) Session Table؟ +
جدول فيه كل الـ Sessions النشطة حالياً داخل الفايروول.
20) ترتيب معالجة الـ Packet داخل الفايروول؟ +
1. Zone Check -> 2. NAT Policy -> 3. Security Policy -> 4. Security Profiles -> 5. Forwarding.
21) ما هو SSL Decryption؟ +
فك تشفير ترافيك الـ HTTPS علشان الفايروول يقدر يفحص المحتوى والملفات اللي جواه.
22) SSL Forward Proxy vs Inbound Inspection؟ +
Forward Proxy: لفحص خروج المستخدمين للإنترنت.
Inbound Inspection: لفحص الترافيك الداخل للسيرفرات الخاصة بك.
23) DoS Protection؟ +
يحمي من الـ Flood Attacks.
24) Zone Protection؟ +
يحمي الـ Zone نفسها من (SYN Flood, Port Scan, ICMP Flood).
25) URL Filtering vs DNS Security؟ +
URL Filtering: فلترة المواقع.
DNS Security: يمنع الـ DNS-based attacks و C2 domains.
26) كيف يعمل الـ App-ID بالتفصيل؟ +
Signature Matching, Protocol Decoding, Heuristic Analysis, SSL Decryption (لو مشفر). وبيقوم بعمل Re-evaluation للجلسة لو التطبيق اتغير.
27) Stateful Inspection vs Application Inspection؟ +
Stateful: بيتابع حالة الاتصال (TCP Handshake).
Application: بيفهم التطبيق نفسه في Layer 7.
28) كيف يتم إنشاء الـ Session؟ +
أول Packet بتعمل Lookup في NAT و Security Policy، لو مسموح بيتم إنشاء Session وإضافتها للـ Table.
29) كيف يعمل الـ Threat Prevention؟ +
بيفحص الترافيك باستخدام: IPS Signatures, Antivirus, Anti-Spyware, WildFire Integration.
30) Security Profile vs Security Profile Group؟ +
Profile: نوع حماية واحد (مثلاً AV فقط).
Group: مجموعة Profiles متجمعة مع بعضها لتسهيل إضافتها للـ Policy.
31) ما هو Panorama؟ +
منصة إدارة مركزية من Palo Alto Networks لإدارة أكثر من Firewall في نفس الوقت.
32) Device Group vs Template في بانوراما؟ +
Device Group: للسياسات (Policies & Objects).
Template: لإعدادات الشبكة والواجهات (Network & Interface Settings).
33) Commit vs Push؟ +
Commit: حفظ التغييرات على الجهاز المحلي.
Push: إرسال الإعدادات من Panorama للأجهزة البعيدة.
34) ما هو الـ High Availability (HA)؟ +
ربط جهازين Firewall معاً لزيادة التوافر وتجنب انقطاع الخدمة.
35) Active/Passive vs Active/Active؟ +
Active/Passive: واحد شغال والتاني Standby (الأكثر استخداماً).
Active/Active: الاثنين شغالين في نفس الوقت.
36) ما هي الـ HA Interfaces؟ +
HA1: Control Link.
HA2: Session Sync.
HA3: Packet Forwarding (في حالة A/A).
37) كيف يتم الـ Synchronization في الـ HA؟ +
Sync Config, Sync Session Table, و Heartbeat Monitoring للتأكد من حالة الجهاز الآخر.
38) خطوات الـ Troubleshooting لو الترافيك مش بيمر؟ +
1. Check Interface Up/Down
2. Check Zones
3. Check NAT
4. Check Security Policy
5. Check Routing
6. Check Logs.
39) أهم أوامر الـ CLI؟ +
show session all
show routing route
show interface all
test security-policy-match
40) كيف تتبع Session معينة؟ +
1. هات الـ Session ID من Traffic Logs.
2. استخدم الأمر:
show session id
